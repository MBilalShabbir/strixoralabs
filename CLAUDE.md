# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Commands

```bash
npm run dev      # Development server on http://localhost:3000
npm run build    # Production build (also runs TypeScript type-check)
npm run start    # Serve the production build
npm run lint     # ESLint via next lint
```

There are no tests. TypeScript errors surface during `npm run build` — always run it after making changes to confirm a clean compile.

## Architecture

**Next.js 16 App Router.** All pages are in `app/` as server components by default. Client components are opt-in via `"use client"` and are mostly in `components/`.

### Page → Component pattern

Each route (`app/about/page.tsx`, etc.) is a thin server component that:
1. Exports `metadata` for SEO
2. Composes section components in order
3. Uses `dynamic()` from `next/dynamic` for heavy animated sections (`ServicesDetail`, `IndustriesDetail`, `ProjectsGrid`) to split their Framer Motion bundles

Each page has a corresponding `loading.tsx` that renders skeleton components from `components/ui/Skeleton.tsx` while the page loads.

### Component organisation

- `components/layout/` — `Navbar`, `Footer`, `NewsletterForm` (extracted from Footer because it uses `onSubmit`)
- `components/brand/` — `OwlLogo` with three variants: `default` (transparent bg), `contained` (indigo square), `mono` (currentColor)
- `components/home/` — one file per home page section
- `components/{about,services,industries,projects,contact}/` — one file per inner page section
- `components/ui/` — shared primitives: `Button`, `Badge`, `SectionLabel`, `PageHero`, `Skeleton`, `ThemeToggle`
- `components/providers/` — `ThemeProvider` wrapping `next-themes`

### Inner page pattern

All five inner pages use the shared `PageHero` component (`components/ui/PageHero.tsx`) which renders the aurora background, fine grid overlay, bottom fade, and staggered heading animation. Pass `label`, `headline`, and `subheadline` props — do not recreate this background per-page.

### Contact form

`ContactForm` (`components/contact/ContactForm.tsx`) posts JSON to `app/api/contact/route.ts`. The API route:
- Checks a `_honey` honeypot field first and silently succeeds for bots
- Validates with Zod
- Sends via Resend if `RESEND_API_KEY` is set; otherwise logs to console
- No code changes are needed to enable email — just set the env var

### Styling system

**Tailwind v3** with a custom config. Key things not discoverable from Tailwind docs alone:

- `bg-surface`, `bg-surface-1/2/3` — the dark surface depth stack (`#080810` → `#16161f`)
- `.card-surface` / `.card-surface-hover` — CSS utility classes in `globals.css` encoding Level 1/2 card appearance including the `inset 0 1px 0` top-edge highlight
- `.chip` — pill badge with brand border/bg, defined in `globals.css` `@layer components`
- `.glass` — `backdrop-filter: blur(16px)` used by the navbar on scroll
- `shadow-inset-t` / `shadow-inset-t-brand` — Tailwind custom box shadows for the top-edge inset highlight
- `ease-spring` — `cubic-bezier(0.22, 1, 0.36, 1)`, used for all transitions and Framer Motion animations
- `violet-*` colour scale — secondary palette for AI/ML contexts (do not use for structural UI)

**Tailwind `@apply` constraint**: Arbitrary opacity values like `brand-500/18` work in JSX `className` strings but fail inside `@apply`. Use only standard Tailwind opacity increments (`/10`, `/20`, etc.) inside `globals.css`.

### Framer Motion conventions

- Stagger pattern: `staggerChildren: 0.1, delayChildren: 0.05` on the parent `variants` object
- Standard spring easing: `[0.22, 1, 0.36, 1]` stored as `const ease = [...] as const`
- All `whileInView` animations use `viewport={{ once: true, margin: "-60px" }}`
- `useReducedMotion()` is respected — check before complex animations

### SEO / metadata

- `app/layout.tsx` sets `metadataBase`, the template `title`, and the default OpenGraph/Twitter images pointing to `/opengraph-image` (generated by `app/opengraph-image.tsx`)
- Inner pages override `title` and `openGraph.url` / `openGraph.description` — always set `alternates.canonical` on new pages
- `app/sitemap.ts` and `app/robots.ts` are auto-discovered by Next.js at `/sitemap.xml` and `/robots.txt`

### Environment variables

See `.env.local.example`. Only `RESEND_API_KEY`, `FROM_EMAIL`, and `CONTACT_EMAIL` are used. The site is fully functional without any of them.
